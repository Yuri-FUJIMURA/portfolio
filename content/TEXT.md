# テキスト担当ガイド — content/

## ファイル一覧

| ファイル | 内容 |
|---|---|
| `profile.json` | About 欄のプロフィール（名前、経歴、タグ、SNS） |
| `works.json` | Works の全データ（トップのカード + 詳細ページ） |
| `studies.json` | Study の全データ（トップのカード + 詳細ページ） |

## works.json の構造

```json
{
  "id": 0,
  "category": "カテゴリ（英語、大文字表示される）",
  "title": "作品タイトル",
  "cardDescription": "トップページのカードに表示される短い説明",
  "detailDescription": "詳細ページに表示される長い説明",
  "tools": "使用ツール（スラッシュ区切り）",
  "image": "img/works/ファイル名.jpg",
  "links": [
    { "label": "表示テキスト", "url": "https://..." }
  ],
  "images": [],
  "sections": []
}
```

## studies.json の構造

works.json と同じ構造に加えて `venue`（発表場所）フィールドがあります。

## 新しい作品を追加する手順

1. `img/works/` に画像を追加（英語ファイル名で）
2. `content/works.json` の配列末尾にオブジェクトを追加
3. `id` は連番（既存の最大値 +1）
4. GitHubにプッシュ → 自動デプロイ

## 作品の順番を変えたい場合

配列の並び順がそのまま表示順です。オブジェクトの位置を入れ替えてください。
`id` は prev/next ナビゲーションに使われるので、並び替えたら `id` も振り直してください。

## 編集時の注意

- JSON はカンマの付け忘れ・余計なカンマでエラーになる。末尾カンマ（最後の `},` の後に `]`）は NG
- 画像ファイル名に日本語を使わない
- HTML タグ（`<br>` 等）は cardDescription 内で使用可
- ダブルクォート `"` はテキスト内で使えない。使いたい場合は `\"` とエスケープする
